What's happening here?

example-smart-app.js and index.html have been customized to include a crfurl variable populated with 'https://ocform3.mystudy.me/preview/::YYBD?'. This refers to an OC Participate form in preview mode on demo1. It then uses some (very cruddy) javascript to append to the URL a few of the variables pulled from the patient record. The GET parameters of the URL are constructed to map to the relevant OCP Form fields using this method. So, when you click the CRF link, the form opens with the values from the patient's health record pre-populated.

How is the URL constructed?

==> https://ocform3.mystudy.me/preview/::YYBD? <== the base form URL
==> d[/F_PHYSICALEXAM_ENGLISH/IG_PHYSI_UNGROUPED/I_PHYSI_SYSTOLIC]=systolicbp <== the systolic bp, where 'systolicbp' is replaced by the code with the actual value
==> &d[/F_PHYSICALEXAM_ENGLISH/IG_PHYSI_UNGROUPED/I_PHYSI_DIASTOLIC]=diastolicbp <== the diastolic bp, where 'diastolicbp' is replaced by the code with the actual value

Why can't I save the form?

We're using an OC form in preview mode because currently that's the only URL that passes the pre-populated values through to the form. Next step would be to do so with an anonymous form, but that will require a change to the OCP code

What else?

  • Do this with an anonymous form and be able to save the data, as described above
  • Open in a new window or have a redirect back to the smart app, or do it in iframe within the app
  • Add SSID to the OC form an populate it with a patient ID, if such a thing is accessible from Cerner
  • Support multiple forms at once
  • Support using OC4 CRFs (not anonymous forms) and adding some security
  • Support dynamically populating the list of forms based on what the patient needs to have completed
  • Support dynamically (or generically?) generating the smart app based on the OC study design, with mapping between the FHIR elements and OC xform items done as a binding in the form definition.

Secure?

Kind of, but not really. The Cerner resources are protected by oauth, so they are only retrievable by an authorized user. No security on the OC side yet, though.
